### 操作说明书（Code Adventurers）

本说明书面向一线使用者与联调同学，涵盖四类角色操作与常用 API 示例。开发/环境细节请参考《环境配置.md》《开发与调试指南.md》。

---

### 基础信息

- **服务地址**（开发默认）：`http://localhost:3000`
- **鉴权方式**：请求头 `x-user-id: <用户ID>`（种子用户：`student-1`、`teacher-1`、`parent-1`、`admin-1`）
- **重置数据**：`POST /api/system/reset`（仅内存模式有效）
- **遥测**：`GET/POST /api/telemetry`

示例（PowerShell/cmd 均可）：

```bash
curl -X POST http://localhost:3000/api/system/reset -H "x-user-id: admin-1" -i
```

### 学生端（Student）

- **游客登录**（获取新 `userId`）

```bash
curl -X POST http://localhost:3000/api/auth/guest -H "Content-Type: application/json" -d '{"name":"小明"}'
```

- **班级登录**（邀请码加入班级）

```bash
curl -X POST http://localhost:3000/api/auth/class \
  -H "Content-Type: application/json" \
  -d '{"inviteCode":"CA-CLASS-1","name":"新同学"}'
```

- **获取地图**

```bash
curl http://localhost:3000/api/student/map -H "x-user-id: student-1"
```

- **关卡准备信息**（胜利条件/可用积木/教学漫画）

```bash
curl http://localhost:3000/api/student/levels/level-1/prep -H "x-user-id: student-1"
```

- **运行关卡**（提交指令序列）

```bash
curl -X POST http://localhost:3000/api/student/levels/level-1/run \
  -H "x-user-id: student-1" -H "Content-Type: application/json" \
  -d '{"program":[{"type":"move"},{"type":"collect"}]}'
```

- **提交结算**（星级/步数/时长/回放等）

```bash
curl -X POST http://localhost:3000/api/student/levels/level-1/complete \
  -H "x-user-id: student-1" -H "Content-Type: application/json" \
  -d '{"stars":3,"steps":3,"hints":0,"duration":20,"bestDifference":0,"replayLog":[]}'
```

- **获取提示（Hint）**

```bash
curl -X POST http://localhost:3000/api/student/hints/level-1 \
  -H "x-user-id: student-1" -H "Content-Type: application/json" \
  -d '{"attempts":2,"lastError":"E_COLLIDE"}'
```

- **装扮**

```bash
curl http://localhost:3000/api/student/avatar -H "x-user-id: student-1"
curl -X PUT http://localhost:3000/api/student/avatar -H "x-user-id: student-1" -H "Content-Type: application/json" -d '{"equipped":"starter-cape"}'
```

- **设置与重置进度**

```bash
curl http://localhost:3000/api/student/settings -H "x-user-id: student-1"
curl -X PUT http://localhost:3000/api/student/settings -H "x-user-id: student-1" -H "Content-Type: application/json" -d '{"lowMotion":true}'
curl -X POST http://localhost:3000/api/student/settings/reset-progress -H "x-user-id: student-1"
```

- **沙盒创作与作品库**

```bash
curl http://localhost:3000/api/student/sandbox -H "x-user-id: student-1"
curl -X POST http://localhost:3000/api/student/sandbox -H "x-user-id: student-1" -H "Content-Type: application/json" -d '{"title":"我的第一个作品","data":{}}'
curl -X PATCH http://localhost:3000/api/student/sandbox/<projectId> -H "x-user-id: student-1" -H "Content-Type: application/json" -d '{"status":"published"}'
curl http://localhost:3000/api/student/library -H "x-user-id: student-1"
curl -X POST http://localhost:3000/api/student/library/<workId>/comment -H "x-user-id: student-1" -H "Content-Type: application/json" -d '{"content":"很棒！"}'
```

### 教师端（Teacher）

确保请求头 `x-user-id: teacher-1`。

- **课程/章节/关卡**

```bash
curl http://localhost:3000/api/teacher/courses -H "x-user-id: teacher-1"
curl -X POST http://localhost:3000/api/teacher/courses -H "x-user-id: teacher-1" -H "Content-Type: application/json" -d '{"name":"新课程","description":"描述"}'
curl -X POST http://localhost:3000/api/teacher/courses/<courseId>/chapters -H "x-user-id: teacher-1" -H "Content-Type: application/json" -d '{"title":"第一章"}'
curl -X POST http://localhost:3000/api/teacher/chapters/<chapterId>/levels -H "x-user-id: teacher-1" -H "Content-Type: application/json" -d '{"level":{"id":"level-x","name":"关卡 X","width":3,"height":3,"tiles":[],"start":{"x":0,"y":0,"facing":"east"},"goal":{"reach":{"x":0,"y":0}},"bestSteps":1,"hints":[],"chapterId":"<chapterId>","comic":"","allowedBlocks":[],"rewards":{"stars":3,"outfit":null}}}'
```

- **编辑关卡**


- **班级管理与布置**

```bash

curl -X PATCH http://localhost:3000/api/teacher/classes/<classId>/hint-limit -H "x-user-id: teacher-1" -H "Content-Type: application/json" -d '{"hintLimit":2}'
```

- **看板与回放**

```bash
curl http://localhost:3000/api/teacher/analytics/progress -H "x-user-id: teacher-1"
curl http://localhost:3000/api/teacher/analytics/heatmap -H "x-user-id: teacher-1"
curl http://localhost:3000/api/teacher/analytics/replays/<studentId>/<levelId> -H "x-user-id: teacher-1"
```

- **作品审核**

```bash
curl http://localhost:3000/api/teacher/works/pending -H "x-user-id: teacher-1"
curl -X POST http://localhost:3000/api/teacher/works/<workId>/review -H "x-user-id: teacher-1" -H "Content-Type: application/json" -d '{"status":"approved"}'
```

### 使用小贴士

- 本地联调默认处于“内存模式”，重置数据后将重新载入种子用户/课程/首关
- 任何需要权限的接口均依赖请求头 `x-user-id` 与角色匹配（student/teacher/parent/admin）
- 运行与结算接口字段需与 `packages/engine/src/types.ts` 的结构匹配（指令、错误码等）
- 若启用 MySQL，请先执行 `docs/database_schema.sql` 与 `docs/mysql_init.sql`

---

### 常见问题排查（精选）

- `401 未登录`：缺少 `x-user-id` 或用户不存在
- `403 无权限`：用户角色与接口不匹配
- `关卡不存在/班级不存在`：请求参数 ID 错误或数据未初始化
- `ERR_MODULE_NOT_FOUND`（本地开发）：确认已 `npm install` 且使用 `npm run dev` 启动
