### 项目总览（Code Adventurers）

本项目是“代码奇兵 Code Adventurers”的教学编程平台雏形，包含学生、教师、家长与管理员四端闭环的后端 API、最小化前端组件示例，以及可复用的关卡执行引擎（VM）。

---

### 仓库结构

- **apps**: 前端示例组件
  - `apps/student` 学生端 UI 组件示例
  - `apps/teacher` 教师端 UI 组件示例
- **packages**: 可复用包
  - `packages/engine` 课程/关卡执行引擎（LevelSimulator、Hint 计算等）
- **services**: 后端服务
  - `services/api` Express API，内存/MySQL 双实现的数据上下文
- **levels**: 示例关卡与数据
- **scripts**: 本地开发脚本（`dev-server.ts` 等）
- **tests**: Jest 测试，覆盖 engine 与 API

路径别名（TS）：`@engine/*`、`@api/*`、`@student/*`、`@teacher/*` 指向对应源码目录。

---

### 技术栈与构建

- 语言与运行时：TypeScript（严格模式）、Node.js ESM
- Web 服务：Express 4
- 数据：内存实现 + MySQL 8（JSON 列，生成列）
- 测试：Jest + Testing Library
- 开发命令：
  - `npm run dev` 启动 API（默认端口 3000）
  - `npm run build` 使用 `tsc -b` 编译至 `dist/`
  - `npm test` 运行测试

详见 `docs/开发与调试指南.md` 与 `docs/环境配置.md`。

---

### 模块职责

- `packages/engine/src/index.ts`
  - `LevelSimulator` 执行指令序列、输出步数/星级/日志
  - `computeHint` 根据错误/尝试次数给出渐进式提示
  - `types.ts` 定义关卡与指令类型（方向/条件/错误码等）

- `services/api/src/index.ts`
  - 身份/鉴权（请求头 `x-user-id`）
  - 学生端：地图、关卡运行/结算、装扮、设置、沙盒与作品库、Hints
  - 教师端：课程/章节/关卡、班级布置、分析看板（热力图/回放）、作品审核
  - 家长端：孩子列表、进度、周报
  - 管理端：概览、素材、账号、数据导出
  - 系统：`/api/system/reset` 重置内存数据；`/api/telemetry` 遥测缓冲

- `services/api/src/store.ts`
  - `DataContext` 抽象，`EntityStore<T>` 接口
  - 内存与 MySQL（`mysql2/promise`）实现，自动建表（JSON 列 + 生成列）

- `services/api/src/seed.ts`
  - 种子数据：`student-1`、`teacher-1`、`parent-1`、`admin-1`，课程/章节/首关
  - `ensureSandboxUnlock`：通关章节自动解锁沙盒

- `services/api/src/telemetry.ts`
  - `TelemetryBuffer`：内存事件缓冲、查询与清空

### 核心特性一览

- **学生端（S）**：游客/班级登录、冒险地图、关卡准备/运行/结算、Hints、装扮、图鉴、沙盒创作与班级分享、设置与进度重置
- **教师端（T）**：课程/章节/关卡管理、班级布置与提示次数上限、进度与热力图、回放、作品审核
- **家长端（P）**：孩子进度、周报
- **管理员（A）**：运营概览、素材与账号管理、数据导出

详尽清单与对应代码位置见 `docs/功能清单.md`。

---

### 数据与存储

- 内存模式：默认用于本地开发/测试，`/api/system/reset` 可重置
- MySQL 模式：设置 `MYSQL_URL` 或拆分变量（`MYSQL_HOST/USER/PASSWORD/DATABASE`）即启用
- 表结构：见 `docs/database_schema.sql` 与 `docs/mysql_init.sql`

### 测试与质量

- `tests/packages/engine.test.ts`：指令执行、错误码、Hints
- `tests/services/api.test.ts`：端到端流程（登录→关卡→结算→报表）

### 运行方式（摘要）

1) `npm install`
2) 可选：初始化 MySQL（见环境文档）
3) `npm run dev` 启动 API 并自动载入种子数据
4) 调试请求需设置请求头 `x-user-id`（如 `student-1`）
